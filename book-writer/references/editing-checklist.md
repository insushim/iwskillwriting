# Editing Checklist Reference

전문 4단계 편집 체크리스트. 출판 산업 표준 기반.

---

## 편집 4단계 개요

```
1단계: 발전적 편집 (Developmental Editing) — 구조, 플롯, 캐릭터
2단계: 문체 편집 (Line Editing) — 문장, 문체, 어조
3단계: 교정 편집 (Copyediting) — 문법, 일관성, 사실 확인
4단계: 교열 (Proofreading) — 오타, 포맷, 최종 확인
```

---

## 1단계: 발전적 편집 (Developmental Editing)

### 구조 검증
- [ ] 선택한 구조 프레임워크의 모든 비트/포인트가 존재하는가?
- [ ] 각 막(Act)의 비율이 적절한가?
- [ ] 중간점(Midpoint)이 명확한가?
- [ ] 클라이맥스가 충분히 강렬한가?
- [ ] 결말이 만족스러운가?

### 플롯 검증
- [ ] 인과관계 (A→B→C) 가 논리적인가?
- [ ] 플롯홀이 없는가?
- [ ] 복선(foreshadowing)이 모두 회수되었는가?
- [ ] 미해결 서브플롯이 없는가?
- [ ] 우연의 일치가 과도하지 않은가?

### 캐릭터 검증
- [ ] 주인공의 욕구(want)와 필요(need)가 명확한가?
- [ ] 캐릭터 아크가 완성되었는가? (변화 또는 의미 있는 불변)
- [ ] 적대자의 동기가 이해 가능한가?
- [ ] 모든 캐릭터가 고유한 목소리를 가지는가?
- [ ] 불필요한 캐릭터가 없는가?

### 페이싱 검증
- [ ] 시작이 충분히 빠른가? (첫 10%에 촉발 사건)
- [ ] 처진 구간(saggy middle)이 없는가?
- [ ] 긴장과 이완의 리듬이 있는가?
- [ ] 각 챕터가 다음 챕터를 읽고 싶게 만드는가?
- [ ] 결말이 서두르지 않는가?

### 세계관 검증 (판타지/SF)
- [ ] 마법/기술 체계가 일관적인가?
- [ ] 지리/지명이 일관적인가?
- [ ] 문화/사회 설정이 내적 논리를 따르는가?
- [ ] 설정 덤프(info dump) 없이 자연스럽게 설명되는가?

### 비소설 구조 검증
- [ ] 핵심 주장이 명확한가?
- [ ] 논점의 흐름이 논리적인가?
- [ ] 근거(데이터, 연구, 사례)가 충분한가?
- [ ] 반론에 대한 대응이 있는가?
- [ ] 독자의 실행 가능한 조언이 포함되는가?

## 2단계: 문체 편집 (Line Editing)

### 문장 품질
- [ ] Show, don't tell이 적절히 사용되는가?
  - 나쁨: "그녀는 슬펐다."
  - 좋음: "그녀의 손이 찻잔 위에서 멈췄다. 김이 식어갈 때까지."
- [ ] 감각 묘사가 충분한가? (시각, 청각, 촉각, 후각, 미각)
- [ ] 문장 길이가 다양한가? (단문-중문-장문 혼합)
- [ ] 불필요한 부사/형용사가 제거되었는가?
- [ ] 능동태가 수동태보다 많은가?

### AI 패턴 검출 & 제거
- [ ] 과도한 부사 ("매우", "정말", "완전히", "절대적으로")
- [ ] 클리셰 표현:
  - "심장이 쿵 내려앉았다"
  - "눈물이 볼을 타고 흘렀다"
  - "입술을 꽉 깨물었다"
  - "주먹을 불끈 쥐었다"
  - "숨이 멎을 듯했다"
  - "시간이 멈춘 것 같았다"
  - "가슴이 먹먹해졌다"
- [ ] AI 특유의 패턴:
  - 문단마다 "그러나/하지만"으로 시작
  - 과도한 열거 (A, B, C, 그리고 D)
  - 모든 감정을 명시적으로 설명
  - 매 챕터 시작을 날씨/시간 묘사로
  - 모든 대화 뒤에 감정 태그 ("그가 화나서 말했다")
  - 추상적 비유의 남발
  - "~의 물결/파도/소용돌이"

### 대화문 검증
- [ ] 각 캐릭터의 대화가 구별되는가?
- [ ] 대화가 자연스러운가? (소리 내어 읽기 테스트)
- [ ] 대화 태그가 간결한가? ("said" 위주, 화려한 태그 최소화)
- [ ] 서브텍스트가 있는가? (말하지 않는 것의 의미)
- [ ] 대화가 플롯/캐릭터를 전진시키는가?

### 어조 일관성
- [ ] 작품 전체의 톤이 일관적인가?
- [ ] POV 캐릭터의 어휘가 그 캐릭터에 맞는가?
- [ ] 시대 배경에 맞는 어휘를 사용하는가?
- [ ] 유머의 수위가 일관적인가?

### 반복 검출
- [ ] 같은 단어의 근거리 반복 (같은 문단 내)
- [ ] 같은 문장 구조의 연속 반복
- [ ] 같은 비유/은유의 반복 사용
- [ ] 티 표현(tic words): "갑자기", "결국", "사실"

---

## 3단계: 교정 편집 (Copyediting)

### 문법 & 맞춤법
- [ ] 맞춤법 오류
- [ ] 띄어쓰기 오류
- [ ] 문장부호 오류 (마침표, 쉼표, 따옴표)
- [ ] 외래어 표기법 준수

### 일관성 검증
- [ ] 캐릭터 이름 일관성 (별명, 호칭 포함)
- [ ] 캐릭터 외모 묘사 일관성 (눈 색, 머리 색 등)
- [ ] 나이/생년 일관성
- [ ] 시간 흐름 일관성 (요일, 계절, 연도)
- [ ] 장소/지명 일관성
- [ ] 고유명사 표기 일관성

### 사실 확인 (비소설)
- [ ] 인용 출처 확인
- [ ] 통계 데이터 검증
- [ ] 인명/지명/연도 확인
- [ ] 전문 용어 정확성

### 시제 & 시점
- [ ] 시제 일관성 (과거/현재 혼용 없이)
- [ ] POV 일관성 (시점 이탈 없이)
- [ ] 인칭 일관성

---

## 4단계: 교열 (Proofreading)

### 최종 확인
- [ ] 오타 (특히 동음이의어: 되/돼, 낫다/낳다 등)
- [ ] 누락된 단어/문장
- [ ] 중복 단어 ("것을 것을", "그리고 그리고")
- [ ] 챕터 번호 순서
- [ ] 챕터 제목 일관성
- [ ] 목차와 본문 일치
- [ ] 페이지 번호 (해당 시)
- [ ] 하이퍼링크 작동 (전자책)
- [ ] 줄바꿈/단락 구분 일관성

## 감정 아크 분석 (Emotional Arc Analysis)

### Vonnegut의 스토리 형태 분석
- [ ] 작품의 감정 곡선이 8가지 기본 형태 중 어디에 해당하는가?
  - **Rags to Riches** (상승): 시작 나쁨 → 끝 좋음
  - **Riches to Rags** (하강): 시작 좋음 → 끝 나쁨
  - **Man in a Hole** (하강 후 상승): 좋음 → 나쁨 → 좋음
  - **Icarus** (상승 후 하강): 나쁨 → 좋음 → 나쁨
  - **Cinderella** (상승-하강-상승): 나쁨 → 좋음 → 나쁨 → 좋음
  - **Oedipus** (하강-상승-하강): 좋음 → 나쁨 → 좋음 → 나쁨
  - **Steady Ascent** (꾸준한 상승)
  - **Steady Descent** (꾸준한 하강)

### 챕터별 감정 극성 추적
- [ ] 각 챕터의 감정 극성(positive/negative)이 분석되었는가?
- [ ] 긍정/부정 키워드 빈도가 측정되었는가?
- [ ] 감정 극성 그래프가 생성되었는가?

### 감정 변화 자연스러움 검증
- [ ] 급격한 감정 전환에 충분한 동기가 있는가?
- [ ] 감정 전환의 기울기(gradient)가 자연스러운가?
- [ ] 인접 챕터 간 감정 낙차가 과도하지 않은가? (비의도적)
- [ ] 의도적 감정 충격은 복선으로 뒷받침되는가?

### 절정 배치 최적화
- [ ] 감정 절정(peak)이 구조적으로 적절한 위치에 있는가?
- [ ] 절정 전 충분한 감정 축적이 이루어졌는가?
- [ ] 절정 후 적절한 감정 해소(catharsis)가 있는가?
- [ ] 다중 절정 작품에서 각 절정의 강도가 점진적으로 증가하는가?

## 페이싱 분석 (Pacing Analysis)

### 씬 길이 분포
- [ ] 각 씬의 글자 수/단어 수가 분석되었는가?
- [ ] 씬 길이의 표준편차가 적절한가? (너무 균일하면 단조로움)
- [ ] 중요 씬에 충분한 분량이 할당되었는가?
- [ ] 전환 씬이 지나치게 길지 않은가?

### 액션/대화/묘사 비율
- [ ] 전체 작품의 액션:대화:묘사 비율이 측정되었는가?
- [ ] 장르별 적정 비율에 부합하는가?
  - 스릴러/액션: 액션 40% / 대화 35% / 묘사 25%
  - 로맨스: 액션 20% / 대화 45% / 묘사 35%
  - 판타지: 액션 30% / 대화 30% / 묘사 40%
  - 문학소설: 액션 15% / 대화 30% / 묘사 55%
- [ ] 챕터별 비율이 씬의 성격에 맞는가?

### 긴장도 곡선 (Tension Curve)
- [ ] 갈등 키워드/긴장 표현의 밀도가 추적되었는가?
- [ ] 긴장-이완의 리듬이 규칙적이지 않은가? (예측 불가능해야 함)
- [ ] 긴장도가 전체적으로 점진적 상승 추세인가?
- [ ] 이완 구간이 너무 길지 않은가?

### 처진 구간 (Saggy Middle) 검출
- [ ] 전체 분량의 30~70% 구간에서 새로운 갈등/반전이 투입되는가?
- [ ] 중간 지점에서 긴장도가 시작점 이하로 떨어지지 않는가?
- [ ] 서브플롯이 메인 플롯의 긴장을 보완하는가?
- [ ] 3연속 이상 긴장도 하락 챕터가 없는가?

### 클리프행어 효과 측정
- [ ] 각 챕터 말미에 클리프행어 요소가 있는가?
- [ ] 클리프행어의 유형이 다양한가? (위기/질문/반전/감정)
- [ ] 클리프행어가 다음 챕터 초반에 적절히 해소되는가?
- [ ] 과도한 클리프행어로 인한 피로감은 없는가?

## 한국어 특화 검사

### 번역투 검출 패턴 (영어 직역체)
- [ ] "~하는 것을 발견했다" → "~을 보았다"
- [ ] "그것은 ~였다" (It was ~) → 주어를 구체적으로
- [ ] "그녀의/그의" 과다 사용 → 생략 가능한 소유격 제거
- [ ] "~에 대해서" 남용 → "~을/를", "~에 관해"
- [ ] "~할 수 있었다" (could) → "~했다", "~을 해냈다"
- [ ] "~하기 시작했다" (started to) → "~했다" (직접 서술)
- [ ] "~인 것 같았다" (seemed to) → 단정적 표현 또는 묘사로 전환
- [ ] "~하지 않을 수 없었다" (couldn't help but) → "~했다"
- [ ] "~중 하나였다" (one of the ~) → 구체적 표현

### 한국어 자연스러움 점수
- [ ] 번역투 패턴 빈도가 측정되었는가?
- [ ] 한국어 고유 표현/관용어가 적절히 사용되는가?
- [ ] 조사 사용이 자연스러운가? (을/를 과다, 은/는 부족)
- [ ] 한국어 어순을 따르는가? (수식어-피수식어 순서)
- [ ] 문장 종결 형태가 다양한가? (~다, ~했다, ~였다 반복 방지)

### 존댓말/반말 일관성
- [ ] 서술체의 높임 수준이 일관적인가?
- [ ] 캐릭터 간 호칭/높임이 관계에 맞는가?
- [ ] 같은 캐릭터에 대한 높임 수준이 변하지 않는가? (관계 변화 제외)
- [ ] 내적 독백의 높임 수준이 서술체와 구분되는가?

### 의성어/의태어 활용도
- [ ] 의성어/의태어가 적절히 사용되어 생동감을 주는가?
- [ ] 같은 의성어/의태어가 과도하게 반복되지 않는가?
- [ ] 장면의 분위기에 맞는 의성어/의태어가 선택되었는가?
- [ ] 의성어/의태어가 한국어의 장점을 살리고 있는가?

## Deep POV 검사

### 시점 이탈 검출 (Head-Hopping)
- [ ] POV 캐릭터가 알 수 없는 정보를 서술하지 않는가?
- [ ] 같은 씬 내에서 POV가 전환되지 않는가?
- [ ] 다른 캐릭터의 내면을 직접 서술하지 않는가?
- [ ] "~처럼 보였다", "~인 것 같았다"가 타인의 내면 대신 사용되는가?

### 심리적 거리 (Psychic Distance) 레벨 측정
- [ ] 레벨 1 (가장 먼 거리): 일반적 서술 ("전쟁이 끝난 뒤의 도시는...")
- [ ] 레벨 2: 특정 인물 소개 ("김수현은 그 도시에 살았다")
- [ ] 레벨 3: 행동 묘사 ("그녀는 창밖을 바라보았다")
- [ ] 레벨 4: 생각 진입 ("이제 무엇을 해야 할까, 그녀는 생각했다")
- [ ] 레벨 5 (가장 가까운 거리): 의식의 흐름 ("빌어먹을. 다 끝났어.")
- [ ] 의도하지 않은 심리적 거리 변동이 없는가?
- [ ] 긴장 씬에서 심리적 거리가 가까운가? (레벨 4-5)

### 필터 단어 검출
- [ ] "~을 느꼈다" → 감각을 직접 묘사
- [ ] "~을 보았다/봤다" → 본 것을 직접 서술
- [ ] "~을 들었다" → 소리를 직접 서술
- [ ] "~라고 생각했다" → 생각을 직접 서술 (Free Indirect Discourse)
- [ ] "~인 것을 알았다" → 깨달음을 행동/반응으로 표현
- [ ] "~인 것을 깨달았다" → 깨달음의 결과를 직접 서술

### Free Indirect Discourse 적절성
- [ ] 캐릭터의 생각이 서술에 자연스럽게 녹아드는가?
- [ ] "~라고 생각했다" 없이도 사고가 전달되는가?
- [ ] 캐릭터 고유의 어휘/어조가 서술에 반영되는가?
- [ ] 서술자의 목소리와 캐릭터의 목소리가 적절히 혼합되는가?

## 웹소설 최적화 검사

### 챕터 길이 플랫폼별 적합성
- [ ] 카카오페이지 적정: 3,000~5,000자
- [ ] 문피아 적정: 4,000~6,000자
- [ ] 노벨피아 적정: 3,000~5,000자
- [ ] 리디 적정: 4,000~7,000자
- [ ] 챕터 간 길이 편차가 20% 이내인가?

### 후크/클리프행어 존재 여부 (매 챕터)
- [ ] 첫 문장이 독자의 호기심을 유발하는가?
- [ ] 첫 3줄 안에 갈등/긴장/의문이 제시되는가?
- [ ] 챕터 말미에 다음 화를 읽게 만드는 요소가 있는가?
- [ ] 후크 유형: 위기 예고 / 비밀 암시 / 감정 폭발 / 반전 / 질문 제기

### 첫 3화 몰입도 점수
- [ ] 1화: 주인공과 세계관이 즉시 파악되는가?
- [ ] 1화: 핵심 갈등/목표가 암시 또는 제시되는가?
- [ ] 2화: 주인공의 능력/매력이 드러나는가?
- [ ] 2화: 스토리 진행에 속도감이 있는가?
- [ ] 3화: 독자가 계속 읽어야 할 이유가 확립되었는가?
- [ ] 3화: 장기 목표/미스터리가 설정되었는가?
- [ ] 설정 설명이 스토리 진행을 방해하지 않는가?

### 독자 이탈 위험 구간 검출
- [ ] 3연속 이상 갈등 없는 챕터가 없는가?
- [ ] 긴 설명/묘사 구간 (500자 이상 연속)이 없는가?
- [ ] 새로운 캐릭터가 한꺼번에 3명 이상 등장하지 않는가?
- [ ] 독자가 공감할 수 없는 주인공 행동이 없는가?
- [ ] 지나치게 느린 파워업/성장 구간이 없는가?
- [ ] 반복적인 패턴 (같은 유형의 위기 반복)이 없는가?
